# 开发目标

## 主要目标

构建一个可靠的 Windows 本地编排器，用 Codex 求解课程作业型 C++ 子题，使用 `demo.exe` 验证行为，并把完整求解过程保存在独立的 workspace 中。

## 近期目标

- 保持多个已验证子题上的求解链路稳定
- 让 `demo.exe` 的子题选择变得显式且可重复
- 降低 solver 对猜测格式的依赖
- 提升重试时失败反馈的质量
- 改进对课程特定禁用规则的硬约束检测
- 让回归验证可以稳定地从 checked-in spec 重复执行

## 中期目标

- 在 Windows 上补强 PDF 提取能力，优先接入 `pdftotext`
- 减少对“必须手工知道 demo 参数”这类交接知识的依赖
- 让编码行为在更多作业上都显式且可重复
- 在不破坏正确性的前提下改进 humanizer
- 用更多真实课程样例扩充回归覆盖

## 质量目标

- 在 `5-b15` 和 `5-b16 --sub1..--sub4` 上稳定重复通过端到端验证
- 常见作业的平均重试次数较低
- workspace 产物保持可读、可排障
- 在最终输出前就暴露约束违规
- 最终代码保持正确，同时仍像正常学生作业

## 当前非目标

- 大规模多题批处理
- 完整的自由式多 Agent 架构
- GUI 或 Web 界面
- 超出当前 Windows `g++` 目标之外的广泛工具链兼容性

## 建议的下一阶段重点

1. 在当前 Windows 机器上安装并验证 `pdftotext`。
2. 提高对 PDF 中硬限制和子题特定规则的提取能力。
3. 增加稳定定位可运行 `codex.exe` 的后备策略。
4. 扩充 `5-b16` 当前单一 grouped testcase 之外的回归覆盖。
5. 决定基于编译器参数的字符集转换是否继续保留为长期方案。
